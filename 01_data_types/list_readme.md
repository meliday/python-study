## 리스트 (list)
리스트는 mutable한 자료형이기에, string이나 tuple같은 immutable한 자료형과 달리 method가 그 값을 바꿀 수 있다. 그레서 안바뀌게 하는 매서드가 따로 있음

list의 쓰임새는 무궁무진한 것 같다. 나중에 AI 관련 공부 할 때에도 상당한 활용 능력을 보인다 한다.
set을 활용해 집합 자료형으로 바꾸어 중복을 제거할 때도 많다.


### 써본 메서드
- append 요소 추가하기, 마지막에 요소를 추가할 수 있다. -> 위치 신경쓸 필요 X, 일단 쌓아두자~
- sort 리스트의 요소를 순서대로 정렬해 준다.
- reverse list를 역순으로 뒤집어 준다. 
- index 인덱스를 반환한다. 없으면 오류가 발생한다.
- insert 특정 위치에 특정 값을 삽입한다. -> 여기 정확히 들어가야해, 순서가 의미를 가질때 사용
- remove list에서 첫 번째로 나오는 함수를 삭제한다. 
- pop은 리스트의 맨 마지막 요소를 리턴하고 그 요소를 삭제한다, pop(x)는 x번째 요소를 리턴하고 그 요소를 삭제한다.
- count list안에 x가 몇 개 있는지 조사하여 그 개수를 리턴한다.
- extend(x) x에는 리스트만이 올 수 있으며, 원래 리스트에 리스트를 더한다.


### 헷갈렸던 점
- sort는 숫자와 문자가 섞이면 TypeError가 발생한다. 숫자는 숫자끼리, 문자는 문자끼리
- 리스트의 매서드는 리턴 값이 None인 경우가 많다. String과 달리, 자료형 자체를 바꾸기 때문이다.
- extend와 리스트 더하기(+)는 결과는 비슷해 보이지만, extend는 원본 리스트를 직접 수정하고 +는 새 리스트를 만든다.


#### append와 insert 활용의 차이점
append와 insert는 모두 리스트에 요소를 추가하지만, 사용 의도와 상황이 다르다.

- append(x)
  - 리스트의 맨 마지막에 요소를 추가
  - 가장 자주 쓰이며, 순서만 유지하면 될 때 적합
  - 예시:
    ```python
    lst = [1, 2, 3]
    lst.append(4)
    # [1, 2, 3, 4]
    ```

- insert(i, x)
  - i번째 위치 앞에 요소를 삽입
  - 기존 요소들이 뒤로 밀림
  - 순서가 중요한 경우에 사용
  - 예시:
    ```python
    lst = [1, 2, 3]
    lst.insert(1, 99)
    # [1, 99, 2, 3]
    ```

정리하면, append는 “일단 쌓기”, insert는 “여기 정확히 넣기”에 가깝다.


#### extend와 리스트 더하기(+)의 차이

겉보기 결과는 같아 보이지만, 동작 방식과 의도가 다르다.

- extend
  - 기존 리스트를 직접 늘림 (in-place)
  - 반환값은 None
  - 예시:
    ```python
    a = [1, 2]
    b = [3, 4]
    a.extend(b)
    # a는 [1, 2, 3, 4]
    ```

- 리스트 더하기 (+)
  - 두 리스트를 합친 새 리스트를 생성
  - 원본 리스트는 그대로
  - 예시:
    ```python
    a = [1, 2]
    b = [3, 4]
    c = a + b
    # c는 [1, 2, 3, 4], a는 변하지 않음
    ```

입문 단계에서는 “누적해서 모은다”는 느낌이면 extend,  
“합친 결과가 필요하다”면 + 로 이해하면 충분하다.
