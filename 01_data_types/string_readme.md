## 문자열 (str)
문자열은 immutable한 자료형이기에, list나 dictionary같은 mutable한 자료형과 달리 method가 그 값 자체를 바꿀 수 없다.
즉 바꾸려면, a = a.split() 이런식으로 다시 정의해야 한다.

처음엔 문자열이 그냥 글자인 줄 알았는데, 생각보다 할 수 있는 게 많다.
F-string? 예전에 공부할 때는 배운적 없는데 새로운 것이 생긴 듯 하다.
요즘 코드를 짤 때는 F-string 위주로 사용된다고 한다.
.format은 과도기였고, 읽을 줄만 안다고 하면 된다.
%d %s와 같은 문자열 포멧코드는 C 언어에서 넘어온 걸까? 궁금하다.


### 써본 메서드
- upper(): 전부 대문자로 바꿔줌
- lower(): 전부 소문자로 바꿔줌
- split(): 문자열을 나눠서 리스트로 만들어줌
- replace(): 특정 문자열을 다른 문자열로 바꿈

### 헷갈렸던 점
- input()으로 받으면 무조건 문자열이다
- 그래서 숫자처럼 계산이 안 된다
- split() 안 쓰면 인덱싱이 바로 안 된다



#### upper, lower은 사용자 입력이 제멋대로일 때 - 비교하기 쉽게 “정규화”할 때 필수적으로 이용하게 된다
